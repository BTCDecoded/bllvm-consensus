# Documentation Organization

This document describes the organization of documentation in `bllvm-consensus`.

## Current Structure

### Core Documentation (Root Level)
- **COMPREHENSIVE_MATHEMATICAL_GUARANTEES_REPORT.md** - Primary reference for all mathematical guarantees
- **MATHEMATICAL_SPECIFICATIONS_COMPLETE.md** - Formal mathematical specifications
- **CONSENSUS_COVERAGE_ASSESSMENT.md** - Coverage assessment
- **EDGE_CASE_COVERAGE_AUDIT.md** - Edge case coverage audit
- **EXACT_VERIFICATION_COUNTS.md** - Exact verification counts
- **VERIFICATION.md** - How to verify guarantees
- **PROTECTION_COVERAGE.md** - Protection mechanisms overview
- **MATHEMATICAL_PROTECTIONS.md** - Mathematical protections
- **TEST_DATA_SOURCES.md** - Test data sources
- **REFERENCE_NODE_RPC_TESTING.md** - Reference node testing

### Historical Documentation
- **history/** - Historical implementation milestones (archived)
- **plans/** - Historical planning documents (archived)
- **fixes/** - Bug fix documentation

## Documentation Principles

1. **Timeless**: Documentation should not contain dates, status updates, or temporary information
2. **Self-Referential**: Documentation should reference other docs, not external status
3. **Imperative**: Write in present tense, describing what exists
4. **Minimal Duplication**: Avoid repeating information across multiple documents
